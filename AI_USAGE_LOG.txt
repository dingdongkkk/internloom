I used Claude Opus and Codex as development assistants during the InternLoom hackathon. Claude Opus helped with the initial architecture, normalized database model, JWT/OTP authentication flow, weighted matching algorithm, application and listing state machines, concurrency strategy, and first-pass API scaffolding. Codex was used to review the implementation against the problem statement, inspect the repository, identify missing submission details, improve the README and API contract, add the consolidated schema and dependency corrections, and validate the result with the automated smoke suite and Postman collection checks. I made the final engineering decisions, adapted the generated ideas to the InternLoom requirements, reviewed the business rules, and verified edge cases including duplicate applications, applicant-cap races, withdrawal reopening, authorization isolation, structured errors, notifications, rate limiting, and audit logging. The final code and documentation were tested and reviewed before submission.
